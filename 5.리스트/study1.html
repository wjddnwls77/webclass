리스트

홈페이지에서 구조와 형태가 비슷한 연관이 되어 있는 컨텐츠를 표기하는 방법

li태그는 단독적으로 사용할 수 없다. 반드시 순서의 흐름을 고려하여 순서가 있다면 부모태그에 ol태그를 없다면 부모태그에 ul태그를 사용해야 한다.

ol/ul태그의 자식태그로는 li 태그만 사용할 수 있다.

li태그 안에는 또 다른 문맥들이 형성 될 수 있다. 그동안 배운 제목, 설명태그 포함해서 여러가지 태그문맥이 li태그 안에 사용 될 수 있다.

orderd list - ol

연관된 컨텐츠 즉 리스트로 표현되는 컨텐츠 사이에 순서의 영향이 있는 경우 ol태그를 부모태그로 사용한다.

  1. git add .
  2. git commit -m
  3. git push origin master
  1. 물을 끌인다
  2. 스프를 넣는다
  3. 면을 넣는다

unordered list - ul

연관 되어 있는 컨텐츠 즉 li태그들이 서로 순서의 영향이 없을 때 부모태그로 ul태그를 사용한다.
